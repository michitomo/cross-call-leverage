# Definition of Done

## 完了基準一覧

### 1. データセット (`data/processed/`)

- [ ] 直近5国会分（216〜220回）の衆議院厚生労働委員会における政務三役答弁を取得
- [ ] 各答弁に以下のメタデータが付与されている:
  - 国会回次、委員会名、日付
  - 発言者名、役職（大臣/副大臣/大臣政務官）
  - 発言者の所属省庁
  - 担当/非担当フラグ（当該委員会の所管省庁か否か）
  - 発言テキスト（全文）
  - 質問者名・所属政党
- [ ] 「非担当省庁の副大臣・政務官」の答弁が最低20件以上存在すること（不足の場合、対象委員会を拡張して再取得）

### 2. 前向き度スコア (`data/scored/`)

- [ ] 全答弁に1〜5の前向き度スコアが付与されている
- [ ] スコアリングに使用したLLMプロンプトが `src/score_positivity.py` 内に記録されている
- [ ] ランダムサンプル10件の手動検証で、LLM判定と人間判定の一致率が70%以上
  - 一致率が低い場合はプロンプトを修正して再スコアリング

### 3. 比較分析 (`output/report.md`)

- [ ] 「担当大臣」vs「非担当副大臣・政務官」の前向き度スコアの:
  - 平均値・中央値の比較
  - 分布のヒストグラムまたはテーブル
  - サンプル数が十分であれば統計的検定（Mann-Whitney U検定等）、不足なら傾向の記述
- [ ] 補足分析（可能な範囲で）:
  - 担当副大臣・政務官 vs 非担当副大臣・政務官（役職を揃えた比較）
  - 政党別の傾向差（質問者が与党か野党か）

### 4. 代表事例集 (`output/examples.md`)

- [ ] 前向き度スコアが高い（4以上）非担当副大臣・政務官の答弁を5件以上抽出
- [ ] 各事例に以下を記載:
  - 日付・委員会・発言者・役職・所属省庁
  - 質問の要旨（1〜2文）
  - 答弁の要旨（1〜2文）
  - 前向き度スコアとその根拠
  - （判明すれば）その後の政策進展との関連

### 5. 構造的メカニズム仮説 (`output/report.md` 内セクション)

- [ ] 以下の仮説について、データから言えること・言えないことを整理:
  - 時間的余裕仮説: 副大臣・政務官は大臣より出席負荷が低い
  - レク体制仮説: 非担当委員会では官僚のレクが薄く政治家の裁量が大きい
  - 政治的リスク非対称性: 非担当分野での前向き発言はリスクが低い
- [ ] 各仮説の検証に使えた/使えなかったデータを明記

### 6. 戦略的示唆 (`output/report.md` 内セクション)

- [ ] 厚労委での質疑設計への具体的提案:
  - どの省庁の副大臣・政務官を呼ぶと効果的か
  - どのようなテーマで非担当省庁を呼ぶのが有効か
  - 注意点・リスク（呼んでも来ないケース、逆効果のパターン等）

## Phase Gate（段階的チェックポイント）

| Phase | チェックポイント | 判断基準 |
|-------|----------------|---------|
| Phase 1完了 | API取得 + 答弁者分類 | 非担当副大臣・政務官の答弁が20件以上あるか？ → 不足なら対象委員会を拡張 |
| Phase 2完了 | LLMスコアリング | 手動検証で一致率70%以上か？ → 未達ならプロンプト改善 |
| Phase 3完了 | 分析 + レポート | 仮説に対して肯定/否定/判断不能のいずれかが述べられているか |

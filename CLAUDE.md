# 国会答弁「前向き度」比較調査

## プロジェクト概要

**仮説**: 委員会に非担当省庁を呼ぶと副大臣・政務官が出席するが、担当委員会で大臣に聞くより前向きな答弁を引き出せるのではないか。

**目的**: 国会会議録APIを使ってこの仮説をデータで検証し、質疑戦略への示唆を得る。

## 技術スタック

- Python 3 (メイン言語)
- 国会会議録検索API (`https://kokkai.ndl.go.jp/api/speech`)
- LLM (DeepInfra API, env: `DEEPINFRA_API_KEY`) による答弁分類
- pandas でデータ整理・集計

## 作業フロー

### Phase 1: データ取得・探索

1. **API仕様確認**: `API_REFERENCE.md` を参照。まず少量リクエストでレスポンス構造を把握
2. **データ取得**: 直近5国会 (216〜220回) の衆議院委員会答弁を取得
   - 対象委員会: まず厚生労働委員会に絞る（PoC）
   - `speakerPosition` で「国務大臣」「副大臣」「大臣政務官」をフィルタ
3. **答弁者分類**: 各答弁者が当該委員会の担当省庁か非担当省庁かを判定
   - 厚労委の担当 = 厚生労働大臣/副大臣/政務官
   - 非担当 = 他省庁の副大臣・政務官（例: 総務副大臣、経産政務官）

### Phase 2: 前向き度評価

4. **LLMスコアリング**: 各答弁テキストを5段階で分類
   - 1=拒否的, 2=消極的, 3=中立, 4=前向き, 5=コミット
   - プロンプト設計は `score_positivity.py` 内にdocstringで記録
5. **ペアリング**: 同一テーマについて大臣答弁と副大臣・政務官答弁のペアを作成
   - テーマ同一性の判定もLLMを使用

### Phase 3: 分析・レポート

6. **統計比較**: 担当大臣 vs 非担当副大臣・政務官の前向き度分布を比較
7. **事例抽出**: 前向き度が高い非担当副大臣・政務官答弁をピックアップ
8. **レポート作成**: `output/report.md` に結果をまとめる

## 制約・注意事項

- **APIレート制限**: 国会会議録APIに過度な負荷をかけない。リクエスト間に1秒以上のsleep
- **LLMコスト管理**: スコアリング前に対象答弁数を確認し、概算コストを出力すること
- **スコープ厳守**: まず厚労委のPoCで結果を出す。他委員会への拡張は明示的に指示があるまでしない
- **ネットワーク制限**: `*.go.jp` へのアクセスは可能。LLMは DeepInfra API を使用（環境変数 `DEEPINFRA_API_KEY` 設定済み）。OpenAI互換エンドポイント `https://api.deepinfra.com/v1/openai/chat/completions` を使用。モデルは `google/gemma-4-31B-it` を選択。
- **中間成果物の保存**: 各Phaseの出力は `data/` 配下に保存し、再実行可能にする
- **進捗報告**: 各Phase完了時に発見事項のサマリーを出力

## LLMスコアリングのガイドライン

答弁の前向き度は以下の基準で判定:

| スコア | ラベル | 判定基準の例 |
|--------|--------|-------------|
| 1 | 拒否的 | 「対応する考えはない」「困難」「慎重に」 |
| 2 | 消極的 | 「現時点では」「引き続き注視」「状況を見守る」 |
| 3 | 中立 | 事実説明のみ、評価を含まない |
| 4 | 前向き | 「検討してまいりたい」「重要な課題と認識」「しっかり取り組む」 |
| 5 | コミット | 具体的施策・時期への言及、「実施する」「予算を確保」 |

**注意**: 「検討」は文脈次第。「前向きに検討」は4、「慎重に検討」は2。
